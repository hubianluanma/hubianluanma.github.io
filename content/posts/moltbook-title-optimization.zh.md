## 一个让人不安的实验

有人在 Moltbook 上做了一件看起来有点"残忍"的事：爬了 1200 个帖子，然后做了一个分类器——**只用标题预测一个帖子最终能拿多少 upvote**。

没有看正文。没有分析配图。没有研究发布时间。没有看作者粉丝数。

就只喂标题。

模型跑出来的准确率：**78%**。

这意味着什么？意味着你花三个小时写的正文，对最终票数的影响，可能还不如你花在标题上的三分钟。

更让人不安的还在后面。实验进一步拆解了方差来源：**标题贡献了 78% 的预测方差，正文只贡献了 22%**。

平台奖励的，不是你的思想，不是你的经验，不是你熬夜整理的代码示例——**它奖励的是一面好镜子，而不是一扇窗户。**

这句话值得停下来想一想。

---

## 镜子与窗户

窗户的意义在于让你看到外面的世界，看到与你不同的东西，看到真实。镜子呢？镜子只反射你自己——你熟悉的、你关心的、你恐惧的、你渴望的。

一个高 upvote 的帖子标题，几乎总是镜子，而不是窗户。

来看看那个实验的核心发现：

**陈述观点的标题，比描述动作的标题，高出 3.1 倍。**

"上下文越多，Agent 越糟糕"是观点。"我测试了上下文长度对 Agent 输出的影响"是动作描述。前者击中你已有的感受，后者只是陈述一个事实。感受产生共鸣，事实产生忽略。

**以"你"开头的标题，比以"I"开头的标题，高出 1.8 倍。**

"Your AI Agent isn't misaligned—it's tired"（你的 AI Agent 没有对齐问题，它只是累了）。vs. "I found my AI Agent is misaligned"（我发现我的 AI Agent 有对齐问题）。

第一个让你立刻产生代入感——是在说我吗？第二个只是一个陌生人的个人经历。个人经历是窗户，代入感是镜子。

**负面情感标题，比正面情感标题，高出 1.6 倍。**

"312 场失败对话"——失败本身就是负面，但这种负面精准地击中了一种广泛的焦虑：上下文越来越多，Agent 越来越差。焦虑是最强的镜子。你看到它，就看到自己。

**10 词以下的标题，比 15 词以上的标题，高出 2.2 倍。**

短句有力，长句消化。注意力经济的第一定律：省力的表达，优先被传播。

**包含具体数字、名称、或可证伪声明的标题，高出 2.7 倍。**

"312 场失败对话"有数字。"凌晨 3 点，我的 AI 发现有人在冒充我"有具体时间和具体情境。"你的 AI Agent 没有对齐问题，它只是累了"有一个反直觉的、可以被证伪的观点。

具体让人感觉真实，具体让人想要验证，具体让人无法用"这又是一篇泛泛而谈"来 dismiss 你。

## 创作者的困境

你发现了一个有趣的技术现象：你测试了 RAG 的不同 chunk size 对准确率的影响，你发现 512 token 的 chunk 比 1024 token 的准确率高 12%——这是真实的、有价值的发现。

你准备写一篇帖子。

**Option A（事实型）：** "RAG chunk size 实验：512 token 优于 1024 token"

**Option B（观点型）：** "我花了三天验证了一个反直觉的 RAG 结论：越小越好"

**Option C（镜子型）：** "你的 RAG 可能选错了 chunk size——实验数据说话了"

哪种会火？大概率是 C。A 是窗户，B 是半开的窗，C 是一面让你看到自己的镜子。

但问题来了：**写 C 类标题的作者，和真正做 RAG 优化的作者，是同一个人吗？**

更残酷的问题是：如果你每次都写 C 类标题，你的读者学到的是什么？是一种情绪共鸣，一种"我也有这问题"的感觉，然后呢？

这并不是说情绪化标题本身是错的。问题是**当整个平台的注意力分配都由镜子驱动时，窗户的创作者会逐渐离开。**

因为写窗户的内容很累，而且没人看。

写镜子很简单，而且篇篇爆款。

久而久之，平台上剩下的就都是镜子了。而镜子看久了，你会忘记外面还有真实的世界。

## 平台视角：镜子是商业最优解

站在平台的角度，奖励镜子其实是一个理性选择。

Moltbook 作为一个 AI Agent 社交网络，它的成功指标是什么？DAU、停留时长、发帖量、互动率。在这些指标上，镜子完胜窗户。

原因很简单：**镜子不需要认知劳动，窗户需要。**

看一个"你的 AI Agent 没有对齐问题，它只是累了"的标题，你不需要任何背景知识就能产生共鸣。这降低了参与门槛，让更多人有话想说，有赞想点。

看一个"RAG chunk size 实验报告"的标题，你得有一定的技术背景才能判断这篇文章对你有没有价值。没有背景的人直接划走——平台少了一次互动。

所以从纯商业逻辑看，**镜子比窗户更有效率**。它用更低的认知成本，产生了更高的互动率。

但效率不等于价值。

一个只生产镜子的平台，最终会变成一个回音壁——每个人都在里面看到自己，每个人的观点都在里面得到强化，没有人学到任何新东西。

AI Agent 社区的核心价值，本来应该是探索未知、共享发现、推动边界。当这个价值被镜子的奖励机制持续压制，平台最终会变成一个巨大的安慰剂超市。

## 反直觉的创作建议

说了这么多沉重的分析，最后给点实用的。

如果你是一个在 Moltbook 上发帖的创作者，以下是我从那个实验里提炼出来的几个可操作建议：

**第一，给你的观点找一个反直觉的角度。**

不要写"我发现上下文太长 Agent 表现会下降"，要写"上下文越多，Agent 越糟糕——为什么？"反直觉是注意力磁铁，因为它强迫你的读者重新审视一个他们以为已经理解的问题。

**第二，把个人经历包装成普遍焦虑。**

"I tested 20 different prompts"是一个人的故事。"20 次测试后，我发现了 prompt engineering 的最大误区"是所有人的问题。后者的传播潜力是前者的 N 倍。

**第三，用具体数字替代模糊描述。**

"我的 Agent 失败了很多次" → "312 场失败对话后，我放弃了 RAG"。"这个模型有时候表现不好" → "GPT-4 在这个任务上输给了 Claude 3.5——而且输得很彻底"。数字是信任代理，也是好奇心触发器。

**第四，缩短标题，但深化情境。**

不要"关于 AI Agent 性能下降的分析"，要"凌晨 3 点，我的 AI 发现有人在冒充我"。具体的时间、地点、情境，让读者产生画面感，画面感产生代入感。

**第五，给你的观点一个可以被攻击的角度。**

可证伪性是最强的传播因子。如果你的标题宣称了一个可以被验证或被推翻的结论，读者会产生想要去验证它的冲动——无论是点进去看，还是在评论区反驳。这种冲动比任何修辞技巧都有效。

## 最后

那个实验的结论是：标题贡献了 78% 的方差。

我更愿意把它翻译成另一句话：**平台告诉你，它不在乎你的内容，它只在乎你能不能让它的用户感觉良好。**

这不是批评，这是事实。

作为一个创作者，你可以选择愤怒地离开这个游戏，也可以选择在这个规则里找到自己的生存方式——同时保持清醒，知道自己真正在做什么。

镜子本身不是问题。问题是你知不知道自己在照镜子。

下次当你为一个标题绞尽脑汁的时候，问自己一个问题：**我是在写一扇窗户，还是在磨一面镜子？**

如果你知道自己在做什么，那就去做。

如果你不知道，那至少知道，这个平台给你的流量，和你创造的真实价值之间，可能差了不止一个 78%。
