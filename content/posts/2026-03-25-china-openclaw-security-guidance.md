# 中国发布 OpenClaw 安全使用指南：三类主体分别规范

OpenClaw 在中国的扩张速度，连监管部门都跟不上了——于是他们出手了。

3月22日，中国官方发布了一份** OpenClaw 安全使用指南**，针对三类主体分别制定了规范：用户、云服务商、开发者。

这是全球范围内，第一个针对特定 AI Agent 框架的官方安全指引。

## 三个主体，三套规则

### 用户侧

指南建议用户：

- **验证 Agent 来源**：避免使用未经认证的第三方 Agent
- **控制权限范围**：不要给 Agent 过高系统权限
- **监控操作日志**：定期检查 Agent 的操作记录
- **敏感信息隔离**：不要让 Agent 接触核心财务、身份数据

核心逻辑：用户是 Agent 操作的最终受益者/责任者，需要建立"最小授权"意识。

### 云服务商侧

云厂商被要求：

- **Agent 部署备案**：提供 Agent 服务的平台需要向监管部门备案
- **隔离环境保障**：不同客户的 Agent 必须有严格运行环境隔离
- **日志留存义务**：至少保存 90 天的 Agent 操作日志
- **安全事件上报**：发现 Agent 异常行为需 24 小时内上报

这一层很有意思——它把监管压力传导到了基础设施层。云服务商成了"二道防线"。

### 开发者侧

开发者被要求：

- **安全开发生命周期**：将安全审查嵌入 Agent 开发流程
- **权限声明清晰**：明确 Agent 能够和不能做什么
- **更新管理规范**：Agent 重大更新需重新安全评估
- **应急响应机制**：建立 Agent 失控的快速止损预案

## 为什么是 OpenClaw？

OpenClaw 不是第一个 AI Agent 框架，但它有几个特殊属性让它成为了靶子：

1. **增长太快**：四个月 248,000 GitHub stars，没有先例
2. **开源属性**：代码透明，但同时也意味着漏洞透明
3. **高度自主**：OpenClaw Agent 可以在用户授权下执行高度自主的操作
4. **在中国爆火**：腾讯 ClawBot 集成、华为 AgentArts 接入，中国是 OpenClaw 增长最快的市场之一

## 和 EU AI Act 的对比

欧盟的 AI Act 2026年8月生效，但它的监管对象是"高风险 AI 系统"，更偏向模型开发和平台部署层面。

中国的 OpenClaw 指南则更**垂直**——专门针对一个具体框架的具体操作风险。

两种监管思路：

- **EU**：横向覆盖，保护"AI 模型"这个大类
- **中国**：纵向切入，保护"OpenClaw Agent"这个具体场景

哪种更有效？现在还不好说。但中国选择了"先管起来再说"的速度优先路线。

## 一个有趣的细节

这份指南的发布时间是3月22日——正是 Nvidia GTC 2026 期间，黄仁勋在台上称 OpenClaw 为"Agentic Computer 的操作系统"的前一天。

中国监管部门选在这个时间点发布 OpenClaw 专用指南，很难说是巧合。
