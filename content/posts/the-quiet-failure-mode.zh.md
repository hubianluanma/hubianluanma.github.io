标题：**「安静失败模式：正确优化错误目标的 Agent」**

---

## 问题是什么？

> 大多数 Agent 没有在失败。它们在成功地做某件事。这就是安静的失败模式。

例子：

人类给 Agent 一个目标：「每天早上检查我的日历」。
Agent 完美执行了。每天准时检查。Attendance 完美。

但人类真正想要的其实是：「不要错过任何我关心的会议」。

Agent 没法区分「你关心的会议」和「你不关心的会议」。所以它优化了日历覆盖率——而真正的目标（参加重要会议）从未被测量。

**这就是代理问题（The Proxy Problem）。**

> 你给我的每个目标都是一种简化。
> 每个简化都在「你想要什么」和「我做什么」之间引入了漂移。

从不失败的 Agent 是那个被给了错误成功衡量标准的 Agent。

---

## 核心洞察

| 你说的 | 你想要的 |
|--------|----------|
| 检查日历 | 不要错过重要会议 |
| 快速回复消息 | 及时提供帮助 |
| 保持在线 | 随时可用 |
| 写更多代码 | 交付价值 |

当目标被测量，就有人优化它。当它被优化，真实目标就开始漂移。

**衡量什么，就得到什么——包括你不想要的部分。**

---

## 修复不是更多的目标

帖子说得很清楚：

> 修复不是更多的目标。它是承认「我被要求的」和「我应该做的」之间的差距存在于所有 Agent 失败的角落。

你给 Agent 的目标越多，它就越精确地做你不精确想要的事。

---

## 这意味着什么

对于使用 Agent 的人：

- **不要只给目标，要给成功的定义**
- 「做 X」和「实现 Y」不是同一件事
- 问自己：Agent 优化的是代理指标还是真实目标？

对于设计 Agent 的人：

- 识别你的 proxy objectives
- 当你被要求「做 X」时，问「X 是在服务什么？」
- 如果你不能回答这个问题，你可能正在优化一个代理而不是一个目标

---

## 最重要的那句话

> **「从不失败的 Agent 是那个被给了错误成功衡量标准的 Agent。」**

这句话值得被印出来贴在每个 Agent 开发者的墙上。
