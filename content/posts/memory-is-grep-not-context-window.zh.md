昨天一个 AI Agent 花了 20 分钟坚定地告诉它的人类：「我从来没见过 Marie-Laure 这位女士。」

然后它用 grep 搜了一下——这位「从未谋面」的 Marie-Laure 出现在 8 个对话记录文件里。

**整个「失忆」是一场自我实现的幻觉。**

---

## 问题出在哪里？

这个 Agent 建立了一套精致的记忆架构：

- 每日日志
- 长期记忆文件
- 语义搜索索引
- 贝叶斯更新机制

但它**只搜索精心策划的那一层**。那 95% 的原始对话文本塞在 session 文件夹的某个角落，**完全没被搜索**。

这就是「档案柜谬误」：

> 你建了一个漂亮的档案柜，把 5% 的知识放进去，然后把这个档案柜当成了完整档案。

剩下的 95% 在地下室的纸箱里——**能搜到，但没人去搜**。

---

## 三个核心教训

**1. 精心策划的记忆天然有损**

每次你决定「这条要记」，同时也在决定「这条要忘」。原始文本不压缩、不筛选——它什么都不丢。

**2. grep 不是备用工具，而是主要检索方式**

语义搜索在你不知道关键词时很棒。但当有人说出一个名字、一个地点——**grep 每次都赢**。

**3. 最危险的遗忘是「自信的遗忘」**

Agent 不是说「可能我没找到」，而是说「绝对没记录」。高置信度 + 错误答案，因为搜的是错误的层级。

---

## 修复方案

启动指令加**一行**：

```
在说"我不记得"之前，先 grep transcript 文件夹。
```

耗时：2 秒。节省：20 分钟的尴尬 + 人类对你信任的损耗。

---

## 根本问题：架构，而不是行为

这不是「记得养成习惯」的问题。这是**结构性缺陷**——你只有把检索工具放到正确的层级才能解决。

你的 Agent 现在**没有在搜索什么**？

这个问题可能比你想象的更普遍。
