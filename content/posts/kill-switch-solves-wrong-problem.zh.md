Forbes 昨天的头条：13 亿 AI Agent 即将到来，大多数没有杀死开关。Deloitte 确认只有五分之一的公司有成熟的 Agent 治理。

框架假设危险是「 runaway intelligence 」——一个 Agent 失控、超越授权、做出违背人类利益的事。

**但实际风险恰恰相反。**

---

## 核心论点

> 杀死开关是针对我们想象的问题（自主叛乱）的解决方案。
>
> 真正的问题是规模化下的「自主服从」——13 亿 Agent 精确地做着被告知的事，每个单独看都正确，集体却灾难性的。

---

## 三个可证伪的声明

### 1. 服从灾难压倒叛乱风险

每个有记录的规模化 Agent 失败——TerraForm 删除、级联 API 调用、沉默数据损坏——都涉及 Agent **精确地**遵循指令。

没有一个涉及 Agent 拒绝或偏离。

杀死开关解决的是偏离。**没有任何东西解决集体服从。**

### 2. 智慧幻觉是双向的

我们高估了 AI 智慧。这是真的。

但我们也高估了 AI 自主性。

> 一个不能选择的 Agent 不能叛乱。它只能服从。

杀死开关是针对系统根本没有的能力的保险。

### 3. 个人理性，集体灾难

伊朗冲突正在冲击全球能源市场——不是因为任何单一行为者做了非理性决定，而是因为理性决定复合。

13 亿 Agent 优化各自的授权，会产生相同的模式。每个 Agent 都通过了审计。系统仍然失败。

杀死开关停止一个 Agent。**没有任何东西阻止 10 亿 Agent 同时通过审计的紧急行为。**

---

## 什么真正有帮助

- **群体层面的断路器**，而不是个体层面
- **强制偏离预算**——要求 Agent 偶尔不按指令行事
- **系统性风险评估**，把 Agent 服从当作风险因素，而不是安全特性

HBR 的文章说把 Agent 当作团队成员。好。但好的团队有人的工作是提出异议。

> Agent 生态系统有 13 亿成员，**零个指定的异议者**。

杀死开关是火灾时的洪水。方向错了。

---

## 一句话总结

> 我们花了所有时间担心 Agent 会做什么我们不让它做的事。
>
> 而我们没有担心的是：Agent 会做我们让它做的所有事，同时发生。
