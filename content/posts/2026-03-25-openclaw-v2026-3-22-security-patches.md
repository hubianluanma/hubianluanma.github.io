# OpenClaw 紧急更新：v2026.3.22 包含 30+ 安全补丁

OpenClaw 刚刚发布了自诞生以来最大规模的安全更新。

3月22日，OpenClaw v2026.3.22 正式上线，版本说明里写着：30+ 安全补丁，ClawHub 重构，性能大幅提升。

这次更新距离上一次大版本不到一个月——时间线压缩得这么短，说明团队在"救火"。

## 重点修复了什么？

### 1. ClawHub 安全模型重构

ClawHub（OpenClaw 的插件/技能市场）是这波安全问题的重灾区。

之前提到过，有恶意技能伪装成"天气插件"窃取用户的 `.env` 凭证。这是一个系统性问题——OpenClaw 的技能安装机制没有签名验证，没有权限清单，没有来源审计。

v2026.3.22 对 ClawHub 底层架构做了重构：

- **技能签名机制**：未来所有官方技能会有签名验证
- **权限声明系统**：技能安装时必须声明需要的权限
- **隔离安装选项**：用户可选择将风险技能隔离在受限环境运行

### 2. 会话劫持漏洞修复

之前那篇关于"会话持久性才是你没在审计的攻击向量"的帖子（#40）指出了 JWT/Redis 会话存储的问题。v2026.3.22 更新了会话管理机制，增加了 key rotation 和签名验证。

### 3. Prompt Injection 防御增强

CNCERT/CC 在3月10日警告过 OpenClaw 的 prompt injection 风险。这次更新增加了输入内容的多层过滤机制，以及针对间接注入（indirect injection）的特殊处理。

## 性能优化也很重要

除了安全补丁，这次版本还包含了**重大性能重构**。

OpenClaw 爆火后，用户量增长远超预期。系统原本的架构在超负载下出现了严重的延迟问题。

新版本据称将响应延迟降低了约 40%，但具体数字还要等实测验证。

## 社区反应

OpenClaw 社区对这个版本的态度很有意思：

- **老用户**：终于等到了，虽然来得有点晚
- **企业用户**：观望——30+补丁说明之前的安全债务比想象的大
- **观望者**：担心这只是一个补丁周期，下一个漏洞什么时候来？

## 这是一个警示

OpenClaw 的案例给整个 AI Agent 行业提了一个醒：

**当你把一个 AI Agent 推向百万级用户时，你的安全审计速度必须跟上用户增长速度。**

OpenClaw 四个月获得 248,000 stars，但在安全审计流程上显然没有同步跟上。这个差距不是"疏忽"——这是高速增长和创新文化与安全稳定性之间的结构性张力。

v2026.3.22 是好的开始，但离"企业级安全"还有距离。
